## 🚀 快速入门（方法一）

#### 第1步：访问Repository

[👉 点击访问Repository](https://github.com/yuchenwu73/The-15th-Five-Year-Plan-Study)

#### 第2步：选择文件并编辑

1. 点击 `政策文件/`文件夹
2. 选择一个你感兴趣的文件（比如 `01-发展新质生产力.md`）
3. 点击右上角的✏️**Edit按钮**（铅笔图标）
4. 滚动到文件最底部"参与者解读"区域
5. 参照以下模板格式，添加你的解读内容

#### 第3步：**Commit & PR** 

1. 滚动到页面最下方
2. 在Commit message框中填写：`添加对01-发展新质生产力的理解-张三`（使用完整文件名）
3. 选择**Create a new branch**
4. 点击绿色按钮**Propose changes**
5. 下一页继续点击绿色按钮**Create pull request**
6. 提交完成

## 📚 进阶操作-本地Git方式（方法二）

如果你熟悉Git或希望使用本地编辑器，可以选择这种方式。

### 前置准备

#### 1.安装Git

**Windows系统：**

- 访问[Git官网](https://git-scm.com/download/win)下载安装程序
- 双击安装，按照默认选项完成安装
- 打开命令提示符或PowerShell，输入 `git --version`验证安装

**Mac系统：**

```bash
# 使用Homebrew安装
brew install git
```

**Linux系统：**

```bash
# Ubuntu/Debian
sudo apt-get install git

# CentOS/Fedora
sudo yum install git
```

#### 2.配置Git

首次使用Git需要配置用户名和邮箱：

```bash
git config --global user.name "你的姓名"
git config --global user.email "你的邮箱@example.com"
```

### 基础Git操作

#### 克隆Repository

```bash
# 克隆repository
git clone https://github.com/yuchenwu73/The-15th-Five-Year-Plan-Study.git

# 进入repository目录
cd The-15th-Five-Year-Plan-Study
```

#### 创建Branch

```bash
# 创建并切换到新branch
git checkout -b 你的名字-政策主题

# 例如：
git checkout -b 张三-新质生产力
```

#### 编辑文件

使用任意文本编辑器打开政策文件：

- **Windows**：记事本、Notepad++、VS Code
- **Mac**：TextEdit、VS Code
- **Linux**：gedit、vim、VS Code

按照文件中的模板，添加你的理解和思考。

#### 提交更改

```bash
# 查看修改状态
git status

# 添加修改的文件到暂存区
git add 政策文件/你修改的文件.md

# 提交更改
git commit -m "添加对01-发展新质生产力的理解-张三"

# 推送到远程repository
git push origin 你的名字-政策主题
```

#### 创建Pull Request

1. 访问GitHub repository页面：[👉 点击访问](https://github.com/yuchenwu73/The-15th-Five-Year-Plan-Study)
2. 点击Pull requests → New Pull Request
3. 选择你的branch，填写说明
4. 点击Create pull request，等待审核合并

## 💡 撰写建议

- ✅ **准确理解**：用自己的语言准确表述政策内涵，避免简单照搬原文
- ✅ **专业视角**：结合计算机、网络安全等专业背景进行深度分析
- ✅ **注重实践**：提出具体可行的实践方案，避免空泛表述
- ✅ **言之有物**：注重内容质量，每个部分简明扼要即可

## ❓ 常见问题（FAQ）

**Q：提交后发现内容需要修改怎么办？**
A：可以重新编辑文件提交，或者在Pull Request中进行修改。如果使用本地Git，在原分支上继续修改后重新 `git push`即可，Pull Request会自动更新。

**Q：我可以参与多个政策文件吗？**
A：当然可以！建议每个政策文件创建一个独立branch，分别提交Pull Request，这样便于审核和讨论。

**Q：不熟悉Git操作怎么办？**
A：使用网页直接编辑方式即可，无需本地安装Git，按照[快速入门（方法一）](#-快速入门方法一)的三步操作即可完成。

**Q：我可以评论其他同学的解读吗？**
A：当然可以！鼓励互相学习和交流。可以在Pull Request下方留言，提出建设性意见，点赞优秀的解读。

**Q：忘记创建branch，直接在main上修改了怎么办？**
A：

```bash
# 先把修改暂存
git stash

# 创建新branch
git checkout -b 新branch名

# 恢复修改
git stash pop

# 正常提交
git add .
git commit -m "..."
git push origin 新branch名
```

**Q：多个同学同时修改同一个文件，会不会互相覆盖？**
A：不会覆盖！因为每个同学都是在文件底部**追加**自己的解读，GitHub通常能自动合并这些修改。即使出现冲突，管理员会在合并时手动处理，确保每位同学的内容都保留。建议大家及时提交PR，避免积压。
